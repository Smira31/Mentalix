# Менталикс — Telegram-бот с Mini App

Скелет проекта: `/start` с кнопкой Mini App, полная `/admin`-панель
(медиа-менеджмент + статистика + список юзеров), пустое мини-приложение
в фирменном стиле. Трекеры (настроение, привычки, цели, анализ) добавляем
следующим шагом поверх этого каркаса.

## Структура проекта

```
mentalix-bot/
├── docker-compose.yml     # запускает bot + backend + postgres одной командой
├── .env.example           # пример переменных окружения
├── bot/                   # Telegram-бот (aiogram 3)
├── backend/                # REST API для мини-аппа (FastAPI)
└── frontend/               # мини-приложение (React + Vite + Tailwind)
```

---

## Шаг 1. Получить Bot Token

1. Открой в Telegram **@BotFather**
2. Команда `/newbot`, придумай имя и username (должен заканчиваться на `bot`)
3. BotFather пришлёт токен вида `123456:ABC-DEF...` — сохрани его

## Шаг 2. Узнать свой Telegram ID

1. Открой **@userinfobot** в Telegram, напиши ему что угодно
2. Он пришлёт твой числовой ID — это и есть `ADMIN_IDS`

## Шаг 3. Взять сервер (VPS)

Нужен любой VPS с Ubuntu 22.04+. Дешёвые варианты для старта (от ~300₽/мес):
**Timeweb Cloud**, **Selectel**, **Beget VPS**. Хватит минимальной
конфигурации: 1-2 vCPU, 2GB RAM.

После получения сервера — подключись по SSH и поставь Docker:

```bash
curl -fsSL https://get.docker.com | sh
apt install -y docker-compose-plugin
```

## Шаг 4. Залить проект на сервер и настроить .env

Загрузи папку `mentalix-bot` на сервер (через `scp`, git или FileZilla),
затем:

```bash
cd mentalix-bot
cp .env.example .env
nano .env   # вставь BOT_TOKEN и ADMIN_IDS из шагов 1-2
```

Поле `MINI_APP_URL` пока оставь как есть — заполним после шага 6.

## Шаг 5. Запустить бота и backend

```bash
docker compose up -d --build
```

Проверь логи:

```bash
docker compose logs -f bot
```

Напиши боту `/start` — должно прийти приветствие с кнопкой Mini App
(пока не рабочей — URL ещё не настроен) и `/admin` — должна открыться
админ-панель. Всё это уже работает без мини-аппа.

## Шаг 6. Задеплоить мини-приложение (frontend)

Telegram требует, чтобы Mini App открывался по **HTTPS**-адресу.
Самый быстрый способ получить его бесплатно — **Vercel**:

1. Зарегистрируйся на vercel.com (можно через GitHub)
2. Залей папку `frontend` в свой GitHub-репозиторий
3. На Vercel: **New Project** → выбери репозиторий → в настройках
   укажи **Root Directory: frontend** → Deploy
4. После деплоя Vercel даст адрес вида `https://mentalix-xxxx.vercel.app`

Локально можно проверить перед деплоем:

```bash
cd frontend
npm install
npm run dev
```

## Шаг 7. Подключить Mini App к боту

1. В `@BotFather`: `/mybots` → выбери бота → **Bot Settings** →
   **Menu Button** → **Configure Menu Button** → вставь адрес с Vercel
2. Также обнови `.env` на сервере: `MINI_APP_URL=https://mentalix-xxxx.vercel.app`
3. Перезапусти бота: `docker compose restart bot`

Теперь кнопка «Mini App» в `/start` откроет реальное приложение.

## Как пользоваться `/admin`

- **🖼 Медиа → 👋 Приветствие** — пришли фото/видео/GIF, бот подставит
  его в сообщение `/start` для всех пользователей
- **📊 Статистика** — количество юзеров, кнопка **👥 Юзеры** открывает
  список всех, по клику на юзера — карточка с ID, юзернеймом и фото

## Что дальше (следующие шаги по мере готовности)

1. Трекеры в мини-аппе: настроение (шкала/эмодзи по дням), привычки
   (чек-лист с сериями), недельный/месячный путь (прогресс к целям)
2. Блок «как у вас дела» — короткий чек-ин, сохраняется в БД
3. Автогенерация анализа недели/месяца на основе накопленных данных
   (без ИИ на первом этапе — просто статистика: % выполнения, стрики,
   динамика настроения)
4. Когда дойдём до трекеров — понадобится HTTPS и для backend тоже
   (сейчас работает по http, этого достаточно для пустого мини-аппа).
   Добавим Caddy/nginx с автоматическим SSL, когда мини-апп начнёт
   реально дёргать API
5. Монетизация — платный доступ к курсам/продвинутым трекерам поверх
   готового каркаса (Telegram Stars или внешняя платёжка)

Как будешь готов — скажи, какой трекер добавляем первым (настроение,
привычки или недельный путь), и продолжим от этого скелета.
