# Mentalix — Roadmap

Telegram Mini App для личной системы развития: ритуалы (что делаю) и аскезы (от чего отказываюсь), с ИИ-собеседниками.

## Стек

- **Фронт:** React + Vite + Tailwind, деплой на Vercel (репозиторий Mentalix)
- **Бот:** aiogram (папка bot/), Railway — сервис accurate-expression
- **Backend:** FastAPI (папка backend/), Railway — сервис mentalix-bot
- **База:** PostgreSQL на Railway
- **ИИ:** GigaChat (Сбер)

## Концепция

Приложение строится на двух опорах:
- **Ритуалы** — регулярные действия, которые держат день (что делаю)
- **Аскезы** — сознательный отказ от вредных привычек (чего не делаю)

Плюс ИИ-собеседники трёх типов и аналитика, которая ищет связи между ними.

## Структура экранов

- **Сегодня** — карточка «Сейчас важнее всего», карточка «Считка дня», входы в Ритуалы/Аскезы/Нейротренажёр
- **Мой путь** — цели с wireframe-горами, привязка, прогресс
- **Фокус** — таймер фокус-сессий с визуализацией созвездия-прогресса
- **Аналитика** — проценты выполнения, инсайты, график недели
- **Курсы** — карточки курсов с обложкой, временем прохождения, заметками
- **Mentalix** — три собеседника с раздельной историей (Маяк, Компас, Дневник)
- **Профиль** — статистика
- **Настройки** — профиль+тариф, уведомления, считка дня, связка с сайтом, поддержка, документы, аккаунт

## Готово (до этой сессии)

### Ритуалы
- Отдельная сущность (таблицы rituals / ritual_logs), min/optimal версии, серии с заморозками
- Момент награды: галочка-«поп», свечение, подскок серии, haptic
- Перетаскивание карточек долгим нажатием

### Аскезы
- Отдельная сущность (ascezas / asceza_logs), пять категорий
- Механика «удержался / сорвался», серия чистых дней
- Поля: причина, триггер, замена

### ИИ-собеседники
- Три персоны с раздельными диалогами и системными промптами, голосовой ввод

### Аналитика
- Проценты выполнения, инсайты, корреляции, график недели

### Retention
- Напоминания, еженедельный дайджест, камбэк-уведомления

### Дизайн
- Палитра emerald-deep/emerald/emerald-light, sage, mint, cognac, gold, cream
- Fraunces (заголовки) + Manrope (текст)
- Запрет зума, отключены вертикальные свайпы (в Telegram-контексте)

## Готово (эта сессия)

### Хедер и настройки
- Кастомная кнопка настроек в шапке (шестерёнка на каждом экране)
- Экран настроек: профиль+тариф, уведомления, основные, поддержка, документы, версия, аккаунт
- Нижний навбар переделан под референс Муны: без подложки-«таблетки», прозрачная рамка на капсуле, домик вместо контраста, плоские активные состояния (без подъёма), центральная кнопка — скруглённый квадрат
- Фон приложения — кастомное изображение (перекрашенная абстрактная текстура в бренд-палитре), `setHeaderColor`/`setBackgroundColor` для слитности с Telegram-хедером

### Курсы
- Таблицы courses / course_notes, эндпоинты CRUD + смена статуса + заметки
- Экран со списком (фильтры Все/В процессе/Пройденные), карточка с обложкой и временем прохождения, детальный экран с заметками

### Считка дня
- Таблица user_quotes, эндпоинт «фраза дня» (детерминированный выбор по дате)
- Экран управления фразами в Настройках, карточка на «Сегодня»
- ИИ-инсайт через GigaChat — не сделан, оставлен на будущее

### Фокус-сессии + созвездие
- Таблица focus_sessions, пользователь сам выбирает длительность
- Таймер с выбором 10/25/45 мин, визуализация созвездия (5 точек на одну фигуру)

### Нейротренажёр
- Таблица brain_sessions, 5 типов упражнений: внимание (Струп-тест), память (последовательности), реакция, нейропластичность (слово наоборот), гимнастика (коробочное дыхание)
- Карточка на «Сегодня», статистика по каждому типу

### Подписки и донат
- Таблицы subscriptions, donations
- Бейдж тарифа в Настройках (Базовый/Про), экран сравнения тарифов
- Экран доната с фиксированными суммами — **без реального списания денег**, донат просто пишется в базу

### Веб-версия приложения
- Платформенный адаптер (`src/platform/`) — единый интерфейс поверх Telegram WebApp API и обычного браузера
- Email-вход по одноразовому коду (OTP) для веб-версии — **код пока приходит прямо в ответе API**, реальной отправки писем ещё нет (SMTP/SendGrid не подключены)
- Связка Telegram-аккаунта и веб-аккаунта через одноразовый код, генерируемый в Настройках

## Заметки по эксплуатации

**Деплой фронта:** Vercel, автодеплой по пушу в main (репозиторий Mentalix)
**Деплой бэкенда и бота:** Railway, сервисы mentalix-bot (backend) и accurate-expression (bot), автодеплой по пушу в main (репозиторий mentalix-bot)

**Перед пушем фронта:** прогонять `npm run build` локально — ловит опечатки до того, как упадёт Vercel.

**Перед пушем бэкенда:** прогонять `py -m py_compile <изменённые файлы>.py` из папки `backend/` — ловит синтаксические ошибки до деплоя на Railway. Требует установленного Python (лаунчер `py`, не `python`).

**Миграции:** новые таблицы создаются автоматически при старте (`Base.metadata.create_all`). Новые поля в существующих таблицах добавляются через `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` в `main.py` — этот способ безопасен для повторных запусков.

**.gitignore:** в обоих репозиториях (Mentalix и mentalix-bot) исключены `node_modules/`, `dist/`, `__pycache__/`, `*.pyc`, `package-lock.json` — важно не закоммитить их снова при будущих правках.

**Дубли папок на диске:** на компьютере есть несколько устаревших копий проекта (Desktop, OneDrive) — рабочие версии: `C:\Users\smira\Mentalix` (фронт) и `C:\Users\smira\mentalix-bot` (бэкенд+бот). Остальные копии — старые клоны, не используются.
